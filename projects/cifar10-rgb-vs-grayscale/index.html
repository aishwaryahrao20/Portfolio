CIFAR-10 Cats vs Dogs: RGB vs Grayscale CNN Comparison

A controlled experiment training identical CNN architectures on RGB and grayscale inputs to quantify the information value of color for binary image classification.

Overview

This project trained the same CNN architecture twice on a filtered subset of CIFAR-10 (cats and dogs only) — once on RGB images and once on grayscale — to isolate and quantify the contribution of color to classification accuracy. The design priority was experimental fairness: identical model capacity, identical training pipeline, identical sampled indices, different input channels only.

What I built

  • Designed a three-stage CNN (ConvBlock → MaxPool ×3, Global Average Pooling, MLP classifier) with configurable input channels to support both RGB (3-ch) and grayscale (1-ch) without any difference in depth or capacity.
  • Implemented a fully reproducible pipeline with fixed random seeds across Python, NumPy, PyTorch, and CUDA — ensuring accuracy differences reflect design choices, not randomness.
  • Built separate augmentation transforms per modality: RGB uses ColorJitter for color-space expansion; grayscale uses matched spatial augmentations (RandomHorizontalFlip, RandomCrop, RandomRotation).
  • Filtered CIFAR-10 to cats and dogs using shared index sets, relabeled to binary (0/1), ensuring both experiments train and test on the exact same samples.
  • Trained with AdamW + Cosine Annealing LR + label smoothing for 25 epochs, logging per-epoch loss, accuracy, and total wall-clock runtime for both runs.

Engineering focus

  • The only variable between the two experiments is the number of input channels — everything else is held constant, making the comparison directly controlled.
  • Device-aware execution detects CUDA, Apple MPS, and CPU automatically, enabling cross-platform reproducibility.
  • Both test accuracy and wall-clock runtime are recorded, enabling a cost-benefit tradeoff analysis between RGB and grayscale at inference time.
  • Practically relevant to embedded vision: quantifies whether a grayscale pipeline is "good enough" when compute budget and input resolution are constrained.
Project details
