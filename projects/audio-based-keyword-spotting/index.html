TinyML Keyword Spotting System for Absolutist Language Detection

A complete end-to-end TinyML pipeline deployed on an Arduino Nano 33 BLE Sense — detecting five absolutist language keywords (never, none, all, must, only) from real-time audio using a 7-stage DSP firmware pipeline, an 88,581-parameter MLP trained on 5,775 samples from three merged datasets, and a custom C-weight inference engine achieving 83% validation accuracy and 78–83% real-time embedded accuracy at under 100 ms latency.

Overview

Research has shown that absolutist language — words like "never," "none," "all," "must," and "only" — appears approximately 50% more frequently in anxiety and depression communities than in control forums, making these terms candidate markers for passive ambient mental-health monitoring. This project explores whether a low-power embedded system can reliably spot these keywords in real time, without cloud dependency or a persistent network connection.

The system runs entirely on an Arduino Nano 33 BLE Sense Rev2 (ARM Cortex-M4F @ 64 MHz, 256 KB RAM, onboard PDM microphone). Rather than using the TFLite Micro interpreter, the trained model's weights are exported as a C header file and the forward pass is hand-implemented in Arduino firmware — giving full control over memory layout and arithmetic without interpreter overhead. The full pipeline covers dataset construction from three merged sources, log-energy feature extraction, MLP training with oversampling, and a seven-stage noise-robust DSP preprocessing chain implemented directly in firmware.

Dataset construction

Three separate audio datasets were merged to create a robust training corpus of 5,775 samples across five keyword classes.

  • Google Speech Commands Dataset: Large-scale keyword audio baseline providing substantial coverage for common words and establishing acoustic diversity.
  • ASU Embedded Machine Learning Audio Dataset: A curated dataset aligned to the five target absolutist keywords, providing domain-specific coverage.
  • Manually recorded data (experiment 5/): A minimum of 50 recordings per keyword (250+ total samples) collected personally using the Open Speech Recorder tool and converted from .ogg to .wav using a custom Python script (convert_ogg_wav.py).

Final class counts after merging: all — 1,090; never — 1,178; none — 1,211; only — 1,074; must — 1,222. Total: 5,775 samples. The dataset was split 80/10/10 into train (4,620), validation (577), and test (578) sets. To address class imbalance, the training set was balanced by random oversampling to 993 examples per class, producing a balanced training tensor of shape (4,965 × 101 × 13).

Feature extraction — log-energy frames

Each audio file is loaded at 16 kHz, converted to mono, and padded or trimmed to exactly 16,000 samples (1 second). Rather than computing a full MFCC spectrogram, a lightweight log-energy feature is extracted to match the strict RAM budget of the Cortex-M4. The waveform is divided into 101 overlapping frames; for each frame, a single log-energy value is computed as:

f_t = log(1 + mean(|x|))

All 13 input features per frame are filled with the same log-energy value, producing a (101 × 13) feature matrix per clip — a 1,313-dimensional input after flattening. This design keeps the feature tensor small enough for on-device extraction and inference within 256 KB RAM. Normalization statistics computed from the balanced training set (μ = 0.04995788, σ = 0.05632823) are embedded directly into the Arduino firmware for consistent inference-time z-score normalization.

Neural network architecture and training

A compact 3-layer Multi-Layer Perceptron (MLP) was chosen over a CNN to minimize parameter count and enable full weight export as a C array for manual firmware inference. The architecture processes the flattened 1,313-dimensional input through two ReLU hidden layers before outputting logits over five keyword classes:

  • Flatten → (1,313)
  • Dense(64, ReLU) — 84,096 parameters
  • Dense(64, ReLU) — 4,160 parameters
  • Dense(5, logits) — 325 parameters
  • Total: 88,581 parameters

Trained with Adam optimizer and sparse categorical cross-entropy loss for 25 epochs at batch size 64 on the balanced, z-score-normalized training set. The forward pass (h₁ = ReLU(W₁x + b₁), h₂ = ReLU(W₂h₁ + b₂), z = W₃h₂ + b₃) is re-implemented exactly in Arduino C firmware using the exported weight arrays, with weights transposed to the [output][input] layout expected by the hand-coded matrix multiply.

Firmware DSP pipeline (7 stages)

The Arduino inference sketch implements a seven-stage audio preprocessing chain before running inference, designed to be noise-robust under real-world microphone conditions at PDM gain = 75.

  • DC offset removal: Computes the mean of all 16,000 samples and subtracts it from each sample. Observed DC offsets ranged ±2,000 in testing.
  • High-pass filter (α = 0.997): Single-pole IIR: y[n] = 0.997 × (y[n−1] + x[n] − x[n−1]) — removes slow DC drift while preserving speech content.
  • Low-pass filter (α = 0.80): Single-pole IIR: y[n] = 0.80 × y[n−1] + 0.20 × x[n] — suppresses high-frequency noise.
  • Hard noise gate (threshold = 400): Samples with |x| < 400 are zeroed — eliminates low-level ambient noise between speech events.
  • Silence detection: A clip is rejected if mean(|x|) < 0.001 — prevents silent frames from passing to inference.
  • RMS-based loudness normalization (target RMS = 0.10): scale = 0.10 / RMS, clamped to [0.5, 15]. Observed scale factors: 4.376 and 9.790 — normalizes loudness variation across speakers.
  • Log-energy feature extraction: 101 frame-level log-energy values computed as f_t = log(1 + mean(|x|)), matching the training-side feature pipeline exactly.

Three inference-time enhancements further stabilize predictions: exponential moving average smoothing (p′ᵢ = 0.85 × p′ᵢ + 0.15 × pᵢ), class-specific bias correction offsets, and a detection threshold of p > 0.08. SNR estimation based on the ratio of the loudest-25% to quietest-25% frames produces values of ~24.7–25.9 dB, indicating a stable noise floor.

Measured outcomes

  • Achieved 83% validation accuracy across 5 absolutist keyword classes by training an 88,581-parameter MLP on 5,775 samples merged from three datasets (Google Speech Commands, ASU EML Audio Dataset, and 250+ manually recorded samples), with per-class F1 scores of 0.87 (must), 0.85 (never), 0.85 (only), 0.81 (none), and 0.78 (all) — measured on a held-out 578-sample test split.
  • Achieved 78–83% real-time embedded accuracy at under 100 ms inference latency by hand-implementing the MLP forward pass as a custom C matrix-multiply in Arduino firmware using exported weight arrays — bypassing the TFLite Micro interpreter overhead entirely, validated by live serial output across multiple keyword test utterances at 10–20 cm from the microphone.
  • Enabled noise-robust on-device inference under real-world conditions by implementing a 7-stage DSP preprocessing chain (DC offset removal, high-pass α=0.997, low-pass α=0.80, hard noise gate at threshold 400, silence detection, RMS normalization to target 0.10, and log-energy extraction) followed by EMA probability smoothing (α=0.85) and class-specific bias correction — maintaining consistent SNR estimates of 24.7–25.9 dB across test utterances.
  • Reduced training-set class imbalance impact by oversampling minority classes to 993 examples each (balanced shape: 4,965 × 101 × 13), then applying z-score normalization with training-set statistics (μ=0.04996, σ=0.05633) embedded directly in firmware — validated by consistent per-class recall (0.79–0.90 range) with no single class dominating the confusion matrix.
