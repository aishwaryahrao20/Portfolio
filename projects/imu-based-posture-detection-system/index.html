IMU-Based Posture Detection System with On-Device TinyML

An end-to-end two-phase embedded posture classification system on an Arduino Nano 33 BLE Sense Rev2 — from offline neural network training and activation function benchmarking (Phase 1), to sensor-agnostic 1D-CNN deployment via TFLite Micro on ARM Cortex-M4 with sub-150 ms on-device inference and no cloud dependency (Phase 2).

Overview

This project built a complete posture classification system across two distinct engineering phases. Phase 1 established an offline classification pipeline: the Arduino Nano 33 BLE Sense Rev2 logs 9-axis IMU data at 50 Hz over serial, and a Python training pipeline benchmarks three activation functions — Sigmoid, Tanh, and ReLU — across a two-hidden-layer feedforward neural network, selecting the best-performing variant. Phase 2 deploys posture classification directly on the microcontroller using three independently trained 1D-CNN models — one per sensor modality (accelerometer, gyroscope, magnetometer) — converted to TFLite and embedded as C byte arrays, enabling runtime sensor selection with no host inference dependency.

The target hardware is the Arduino Nano 33 BLE Sense Rev2: ARM Cortex-M4F running at 64 MHz, 256 KB RAM, BMI270 (accelerometer + gyroscope), and BMM150 (magnetometer). Five posture classes are classified: Supine, Prone, Side, Sitting/Standing, and Unknown.

Phase 1 — Offline Neural Network Classification

The Arduino firmware reads raw accelerometer data at approximately 50 Hz and computes four derived features per sample: pitch, roll, dotS (Z-axis gravitational alignment), and dotL (X-axis alignment). A rule-based label (1–5) is assigned on-board for ground-truth annotation and logged over serial to per-posture CSV files across multiple collection trials.

The Python training pipeline (posture_nn.py) loads all CSVs, applies z-score normalization and Gaussian noise augmentation at σ = 0.02 to improve generalization, and trains a two-hidden-layer dense network (128 → 64 neurons) independently across three activation function configurations. Each variant is evaluated with accuracy and loss convergence curves, confusion matrices, and per-class precision/recall metrics.

  • Sigmoid: 99.97% validation accuracy, 99.97% test accuracy.
  • Tanh: 99.98% validation accuracy, 99.98% test accuracy.
  • ReLU: 99.99% validation accuracy, 99.99% test accuracy — fastest convergence, selected as the best model.

All five classes achieved near-perfect precision and recall across every activation variant. ReLU was selected for Phase 2 due to fastest training convergence and highest test accuracy.

Phase 2 — On-Device TinyML Inference (Sensor-Agnostic)

Phase 2 moves inference entirely onto the microcontroller. A separate 9-axis IMU logger streams raw ax, ay, az, gx, gy, gz, mx, my, mz at 50 Hz over serial. A Python dataset builder (build_dataset.py) creates sliding-window datasets per sensor modality — windows of 100 samples × 3 channels with 50% overlap — producing separate .npz archives for accelerometer, gyroscope, and magnetometer data.

Three independent 1D-CNN models are trained, one per sensor modality, using the following architecture:

  • Input: (100, 3)
  • Conv1D — 16 filters, kernel size 5, ReLU
  • Conv1D — 32 filters, kernel size 5, ReLU
  • GlobalAveragePooling1D
  • Dense(32, ReLU)
  • Dense(5, Softmax) — one output per posture class

Each trained model is converted to TFLite via convert_to_tflite.py and exported as a C byte array header for embedding in firmware. The inference sketch (posture_inference.ino) collects a 100-sample window, applies training-set normalization statistics, runs inference within a 32 KB tensor arena, and returns the predicted label and confidence over serial. A Python base station (base_station.py) provides a CLI for runtime sensor selection and live prediction display.

Measured outcomes

  • Achieved 99.99% classification accuracy across 5 posture classes by training a two-hidden-layer dense network (128→64 neurons) with z-score normalization and Gaussian augmentation (σ=0.02) on rule-labelled IMU data, benchmarked across Sigmoid, Tanh, and ReLU activation variants with ReLU demonstrating fastest convergence and highest test accuracy.
  • Deployed 3 sensor-specific 1D-CNN models (~6 K parameters, 19–65 KB TFLite) to an ARM Cortex-M4 via TFLite Micro, delivering sub-150 ms on-device inference with no cloud dependency, within a 32 KB tensor arena — enabled by an automated Keras → TFLite → C-header conversion pipeline.
  • Enabled runtime sensor modality selection (accelerometer, gyroscope, magnetometer) with no firmware recompile, by training and embedding three independently quantized models selectable via the base station CLI, as validated by live prediction matching offline test accuracy in controlled conditions.
  • Reduced dataset sensitivity to sensor noise by applying 50% overlapping sliding windows (100 samples per window) combined with Gaussian augmentation, validated by near-identical per-class precision and recall across all five posture labels and all three activation functions in held-out test evaluation.

System architecture

The two-phase system separates data collection, training, and on-device deployment into well-defined stages. In Phase 1, the Arduino logs derived IMU features over USB serial to CSV; the Python pipeline trains and evaluates multiple model variants offline. In Phase 2, the Arduino logs raw 9-axis IMU frames; the Python pipeline builds sliding-window datasets, trains per-modality CNNs, converts to TFLite, and generates C headers; the firmware embeds all three models and runs inference in real time, streaming predictions to the base station.
