IMU-Based Posture Detection with On-Device TinyML

A resource-constrained posture classification system that converts 9-axis IMU streams into real-time posture decisions on an ARM Cortex-M4.

Overview

This project built an on-device posture detection system using 9-axis IMU data and TinyML deployment. The system classifies five posture classes using normalized, augmented, sliding-window sensor data and runs inference locally on an ARM Cortex-M4 without cloud dependency.

What I built

  • Trained feedforward and 1D-CNN models on 9-axis IMU data for five posture classes.
  • Used z-score normalization, Gaussian augmentation, and 100-sample sliding windows with 50% overlap to increase training robustness.
  • Exported three sensor-specific models through an automated Keras-to-C-header conversion pipeline for embedded deployment.

Measured outcomes

  • Achieved 99.99% classification accuracy across five posture classes.
  • Deployed compact models of approximately 6K parameters and 19-65 KB to an ARM Cortex-M4.
  • Delivered sub-150 ms on-device inference inside a 32 KB tensor arena.
Project details
  • Duration: Aug - Dec 2025
  • Associated with: Arizona State University
  • Hardware: ARM Cortex-M4
  • Model family: Feedforward neural network and 1D-CNN
  • Classes: 5 posture classes
